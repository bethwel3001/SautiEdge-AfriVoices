# Contributing to SautiEdge

Thank you for your interest in SautiEdge. We welcome contributions to help make offline voice AI more accessible and inclusive for East African languages.

By participating in this project, you agree to abide by our Code of Conduct and license terms.

---

## Code of Conduct

We are committed to providing a welcoming, respectful, and professional environment for everyone. We expect all contributors to:
- Use welcoming and inclusive language.
- Be respectful of differing viewpoints and experiences.
- Gracefully accept constructive criticism.
- Focus on what is best for the community.
- Show empathy towards other community members.

---

## How to Contribute

### Reporting Bugs

If you find a bug in SautiEdge, please open an issue in the repository. Include the following details:
- A clear description of the bug.
- Steps to reproduce the behavior.
- Expected versus actual behavior.
- Details of your runtime environment (OS, Python version, dependencies, hardware).
- Error logs or stack traces.

### Submitting Pull Requests

To contribute code or documentation updates, please follow these steps:
1. Fork the repository and create your branch from the `main` branch.
2. If you are modifying code, ensure your changes are fully tested and do not introduce regressions.
3. Write clean, well-commented code following PEP 8 guidelines.
4. Verify that all scripts compile and run without syntax errors.
5. Submit a pull request (PR) describing your changes and referencing any related issues.
